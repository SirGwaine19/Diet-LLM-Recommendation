# CHAPTER 5: REQUIREMENTS

## 5.1 System Requirements

The Diet LLM Recommendation System has been designed as a modern AI-powered web platform that integrates a Python-based backend, an LLM service for natural language understanding, a relational database for structured storage, and a responsive web frontend. To ensure smooth development, dependable execution, and reliable user experience, the system is supported by a clearly defined set of hardware and software requirements. These requirements collectively ensure that every component, from the database engine to the AI processing layer, operates with the necessary performance and stability across both development and deployment environments.

### 5.1.1 Hardware Requirements

The hardware requirements have been selected to support efficient backend execution, AI-driven natural language processing, and responsive frontend rendering. The recommended processor is an Intel Core i5 of the 8th generation or higher, or an equivalent AMD Ryzen 5 processor, which provides sufficient computational power for running the FastAPI backend, executing database operations, and handling concurrent API calls without significant performance degradation. A multi-core processor is particularly important because the backend frequently performs simultaneous tasks such as request handling, database queries, and external LLM API communication.

For memory, the system requires a minimum of 8 GB of RAM, with 16 GB recommended for smoother development and testing, especially when running multiple services such as the backend server, database engine, frontend dev server, and browser-based testing tools at the same time. Adequate RAM ensures that the system remains responsive even when handling extensive meal histories, multiple authenticated sessions, and concurrent API calls. In terms of storage, an SSD with at least 256 GB capacity is recommended to ensure fast read and write operations, quick database query execution, and rapid loading of project dependencies and tooling. SSD-based storage is particularly important during testing and experimentation, where multiple datasets, logs, and database snapshots may need to be accessed frequently.

Although heavy GPU acceleration is not required for the current MVP, since most of the AI processing is offloaded to the cloud-hosted LLM API, an integrated GPU is sufficient for the standard application workflow. However, an optional NVIDIA GTX 1650 or higher GPU can be beneficial during future enhancements that involve local embedding generation, semantic search, or experimentation with custom AI models. Finally, the system also requires a Full HD display, a standard keyboard and mouse, and a stable internet connection. Internet connectivity is essential, as the application relies on cloud-hosted LLM services for natural language understanding and recommendation generation, and uninterrupted access ensures consistent and responsive system behavior.

### 5.1.2 Software Requirements

The software requirements have been carefully aligned with the project’s architecture, technology stack, and deployment expectations. The system supports modern operating systems including Windows 10 or Windows 11 (64-bit), Ubuntu 20.04 or above, and macOS, ensuring that developers and end users can work across diverse platforms without compatibility issues. The choice of cross-platform OS support is essential because the project leverages widely adopted technologies such as Python, Node.js, and PostgreSQL, all of which run reliably on these operating systems.

For the backend implementation, Python 3.9 or higher is required, accompanied by FastAPI for API development, SQLAlchemy as the object-relational mapping framework, Alembic for database migrations, Pydantic for data validation, and python-jose with passlib for secure JWT-based authentication. The backend also depends on an OpenAI Python client to integrate with cloud-hosted Large Language Models such as GPT-4o-mini and GPT-4 for natural language meal parsing and personalized recommendation generation. For the frontend implementation, the system requires Node.js 18 or higher, along with the React framework using TypeScript, the Vite build tool, Axios for API communication, and lightweight state management libraries such as Zustand or React Query for managing UI state and data caching efficiently.

PostgreSQL 14 or higher is required as the primary database engine due to its strong support for relational data models, transaction reliability, and future readiness for advanced extensions such as pgvector for semantic search. Development tools such as Visual Studio Code or PyCharm are recommended for coding, while Git and GitHub are used for version control, branching strategies, and collaborative development. Additional supporting tools, such as Postman or the built-in Swagger UI, are used for API testing, while CSV/Excel tools are used for nutrition dataset preparation and offline analysis. Together, this carefully selected software stack provides a robust, modern, and scalable foundation that supports every aspect of the Diet LLM Recommendation System throughout its development, testing, and deployment lifecycle.

## 5.2 Functional Requirements

The functional requirements of the Diet LLM Recommendation System describe the core operational capabilities that the platform must deliver in order to fulfill its primary purpose of enabling personalized, AI-driven dietary tracking and guidance. Each functional requirement directly corresponds to a real user need, and together they shape a complete system that combines secure access, accurate nutrition computation, intelligent recommendations, and a smooth user experience.

The system includes a robust authentication module that enables secure user registration and login through an email and password mechanism. To maintain protected access throughout the user’s journey, JWT-based session management is implemented, which ensures that every authenticated request is verified before sensitive operations are allowed to proceed. This module not only protects personal information but also forms the trust boundary of the entire platform, guaranteeing that only authorized users can interact with their dietary data and AI-driven recommendations. Closely tied to authentication, the system also provides a profile management module that allows users to update important personal details such as age, weight, height, and activity level. Through the same module, users can configure their dietary preferences, declare allergies, and define personal nutritional targets, which collectively form the foundation for accurate personalization across every other module.

A central functional capability of the system is the meal logging module, which accepts natural language inputs from the user and processes them through a Large Language Model. The LLM extracts structured information about the meal, including food items, quantities, units, and portion sizes, and then maps these items against the curated nutrition database to retrieve accurate nutrient information. This capability is what fundamentally distinguishes the system from conventional diet trackers, as it removes the burden of manual food selection and enables users to log meals using everyday language. The meal logging module is supported by a meal history functionality that allows users to view all previously logged meals along with their detailed nutritional breakdowns. Users can filter their history by date, examine intake patterns over time, and modify or delete entries as needed, ensuring full control over their dietary records.

To complete the intelligent guidance loop, the system also includes a recommendation module that generates personalized daily summaries by comparing the user’s actual nutritional intake with their predefined goals and dietary preferences. By transforming raw nutritional data into meaningful insights, observations, and actionable suggestions, this module elevates the platform from a static tracking application into a dynamic AI-driven dietary assistant. It also captures user feedback on the generated recommendations, allowing the system to evolve over time and continuously improve the relevance of its outputs. Together, these functional capabilities ensure that the Diet LLM Recommendation System delivers a seamless, secure, user-friendly, and intelligent nutrition tracking experience, while also providing a strong foundation for future enhancements such as image-based meal logging, semantic retrieval-augmented recommendations, and multilingual dietary support.

## 5.3 Non-Functional Requirements

In addition to its functional capabilities, the system has been designed to meet several important non-functional requirements that determine its overall quality, reliability, and readiness for real-world deployment. From a performance perspective, the system targets API response times under two seconds for standard endpoints and under five seconds for LLM-driven endpoints such as meal parsing and recommendation generation. The architecture is also designed to support more than one hundred concurrent users without functional breakdown or significant latency degradation, ensuring that the platform remains responsive even as adoption grows.

Security has been treated as a foundational non-functional priority. All user passwords are stored using bcrypt hashing, JWT tokens carry strict expiration policies, all incoming requests undergo schema-based validation, and sensitive communications are routed over encrypted connections. Combined with role-readiness for future administrative access control, these measures ensure strong protection of user data and dietary information. Reliability is supported through structured error handling, connection pooling at the database level, and detailed logging across the backend services, which together enable smooth recovery from transient failures and effective issue diagnosis during maintenance. Usability is supported through clean RESTful API design, automatically generated Swagger documentation, meaningful error messages, and a frontend that is responsive across desktop and mobile devices. Finally, the system has been built with strong emphasis on scalability and extensibility, with a modular architecture that supports future enhancements such as horizontal scaling, advanced AI integrations, and progressive feature expansion. Together, these non-functional requirements ensure that the Diet LLM Recommendation System is not only feature-complete but also dependable, secure, and ready for sustained real-world usage.
