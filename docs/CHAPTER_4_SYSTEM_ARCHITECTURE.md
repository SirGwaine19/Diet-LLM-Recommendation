# CHAPTER 4: SYSTEM ARCHITECTURE

## 4.1 System Architecture Overview

The Diet LLM Recommendation System has been designed using a layered, modular architecture that separates user interaction, business logic, intelligent processing, and data management into independent yet tightly cooperating layers. This architectural approach not only ensures clarity of responsibility across the system but also provides a strong foundation for scalability, maintainability, and future expansion. Each layer is purposefully isolated so that changes in one layer have minimal impact on the others, while the overall system continues to function as a single unified platform delivering personalized dietary intelligence to its users. The architecture is organized into four primary layers, namely the Client Layer, the API Layer, the AI/ML Layer, and the Data Layer.

### 4.1.1 Client Layer

The Client Layer serves as the primary point of interaction between the user and the entire system. It is implemented as a modern, responsive web-based interface that has been carefully designed to focus on simplicity, clarity, and a smooth user experience. Through this interface, the user is able to log meals using natural language descriptions, eliminating the cognitive overhead associated with traditional manual food databases. The interface accepts conversational meal entries such as “two rotis with dal and a glass of milk,” transforms them into a structured request, and forwards them to the backend without requiring the user to perform any technical translation or category mapping.

In addition to meal logging, the Client Layer provides comprehensive views for nutritional summaries and breakdowns. Users can examine the calories, proteins, carbohydrates, fats, and other nutrients associated with each meal, as well as aggregated daily and historical patterns. The interface also delivers personalized AI-generated recommendations directly to the user in a clean, supportive, and easily readable format, ensuring that the system feels like a trusted dietary assistant rather than a static reporting tool. Beyond logging and recommendations, the Client Layer also acts as the central hub for profile and goal management, allowing users to update their personal information, dietary preferences, allergies, cultural food choices, and target nutrition goals. By unifying all these interactions in a single interface, the Client Layer ensures that the user always has full visibility and control over their dietary journey.

### 4.1.2 API Layer

The API Layer functions as the structured communication backbone of the entire system and is implemented as a RESTful backend built using the FastAPI framework in Python. This layer is responsible for receiving requests from the Client Layer, validating their structure and authentication state, orchestrating the underlying logic, and returning well-defined, predictable responses. By following REST principles and exposing clean endpoint contracts, the API Layer ensures that the system remains compatible with both the current web frontend and any future clients such as mobile applications or third-party integrations.

A core responsibility of the API Layer is user authentication, which is implemented using secure JWT-based token management. This includes endpoints for user registration, login, identity verification, and continued session validation, ensuring that all sensitive operations remain protected against unauthorized access. Beyond authentication, the API Layer also handles user profile and goal management, exposing endpoints that allow users to retrieve, update, and personalize their information at any time. Furthermore, this layer is responsible for the meal lifecycle, providing endpoints for natural language meal logging, structured meal storage, history retrieval, filtering, and removal of records when required. Finally, the API Layer manages all recommendation-related interactions, including the generation of personalized AI recommendations, retrieval of past recommendations, and capturing user feedback that can be used to refine future suggestions. Together, these capabilities make the API Layer the central coordinator that ensures secure, reliable, and consistent communication between the user-facing interface and the deeper intelligence of the system.

### 4.1.3 AI / ML Layer

The AI/ML Layer is the most innovative and intelligence-driven component of the entire system, responsible for transforming the application from a traditional dietary tracker into an AI-powered nutrition assistant. This layer integrates advanced Large Language Models such as GPT-4o-mini and GPT-4 to perform deep natural language understanding on user-provided meal descriptions. Rather than relying on rigid keyword matching or fixed templates, this layer interprets meal entries the way a human nutrition expert would, recognizing food items, implicit quantities, preparation methods, and even contextual references such as time of day or meal type. This capability is what allows the system to convert casual food descriptions into precise, machine-readable structured data without requiring effort from the user.

A key responsibility of the AI/ML Layer is the structured extraction of meal data from conversational input. The LLM is guided through carefully crafted prompts to produce reliable JSON-style outputs containing food items, quantities, units, and serving sizes. This output is then handed over to the downstream nutrition calculation pipeline. Beyond extraction, the AI/ML Layer is also responsible for personalized recommendation generation, where the model is given access to the user’s profile, dietary preferences, and recent meal history. Using this rich personalization context, the model produces meaningful, supportive, and goal-aligned recommendations that go far beyond generic dietary advice. To further strengthen the intelligence of this layer, embedding models are also planned to be integrated for semantic food matching as part of the upcoming Retrieval-Augmented Generation enhancement. With this addition, the system will be able to handle vocabulary variations, regional naming differences, and partial descriptions with even higher accuracy, making the AI/ML Layer the long-term core of the project’s intelligence capabilities.

### 4.1.4 Data Layer

The Data Layer forms the foundation of the entire architecture, ensuring that every interaction, profile detail, meal entry, and AI-generated recommendation is reliably stored, retrieved, and protected over time. This layer is implemented using PostgreSQL, a powerful and widely adopted relational database system known for its reliability, transaction integrity, and strong support for structured data. Within this layer, all user-related and dietary-related data is organized into clearly defined tables that follow normalized relational design principles, allowing efficient queries, consistent data relationships, and easy future scalability.

The Data Layer manages multiple categories of data simultaneously. It maintains comprehensive user profile information, including demographic data, dietary preferences, allergies, and personalized nutrition goals. It stores all logged meals along with their parsed structured items and computed nutrient values, allowing users and the system to retrieve detailed historical insights at any point in time. It also maintains a curated nutrition database containing diverse food entries that serve as the reference source for nutrient computation, ensuring that the system can support a wide variety of meals across different cultural contexts. In addition, this layer also stores all AI-generated recommendations and any feedback associated with them, enabling continuous learning and personalization improvements in future iterations. To prepare the system for advanced semantic capabilities, the Data Layer is also designed to integrate a vector store as part of Phase 2, which will support semantic food matching and retrieval-augmented recommendation generation. Through its robust structure, careful organization, and forward-looking design, the Data Layer ensures that the system remains both dependable in the present and ready for advanced data-driven enhancements in the future.

## 4.2 Architectural Significance

By organizing the system into these four well-defined layers, the architecture achieves a strong balance between user experience, intelligent automation, technical reliability, and long-term extensibility. The Client Layer ensures intuitive accessibility, the API Layer guarantees secure and structured communication, the AI/ML Layer delivers human-like understanding and personalization, and the Data Layer provides a stable foundation of accurate and persistent information. Together, they create a complete ecosystem in which conversational meal input is seamlessly transformed into precise nutritional insights and personalized dietary guidance. This layered architectural philosophy not only supports the current implementation of the project but also lays a clear path for future advancements such as image-based meal logging, multilingual support, semantic retrieval, and integration with external health platforms.

## 4.3 Key Features

The Diet LLM Recommendation System has been built around a focused set of intelligent features that together transform a conventional dietary tracker into a personalized AI-driven nutrition assistant. Each feature has been designed not as a standalone utility but as a tightly integrated component that contributes to a smooth, accurate, and supportive user experience. The following subsections describe these key features in detail.

### 4.3.1 Authentication Module

The authentication module is one of the most critical components of the conversational nutrition assistant, as it ensures that only authorized users are able to access the system and interact with their personal dietary information. It verifies the identity of users through secure login mechanisms based on email and password credentials, while remaining extensible to support additional verification methods such as one-time passwords or biometric authentication in future enhancements. By acting as the trust boundary of the entire platform, this module ensures that sensitive personal health information, dietary preferences, allergies, and AI-generated recommendations remain accessible only to the rightful user.

In addition to verifying user identity at the time of login, the authentication module also manages secure session continuity throughout the user’s interaction with the system. This is achieved through JWT-based token management, where every authenticated request is verified before allowing access to protected operations such as meal logging, recommendation retrieval, profile editing, and history browsing. The module is also designed to support role-based access control, enabling clean separation between standard end users and administrative roles whenever the system is extended in the future. Through the use of password hashing, encrypted communication, and standardized authentication protocols, this module strengthens the overall security posture of the system and significantly enhances user trust in the platform.

### 4.3.2 Natural Language Meal Logging

Natural language meal logging is the most distinctive feature of the system and represents a fundamental shift in how users interact with dietary tracking platforms. Rather than forcing users to navigate through long food databases or fill rigid forms, the system allows them to describe their meals in plain everyday language. For example, when a user types something like “I had two eggs, toast with butter, and a glass of orange juice for breakfast,” the LLM-powered parsing engine intelligently interprets the input and converts it into structured machine-readable data. This structured representation captures the individual food items, their quantities, units, portion sizes, and preparation indicators, all without requiring any explicit effort from the user.

This capability dramatically reduces the friction associated with daily diet tracking, especially for users who think and speak about food in conversational terms rather than nutritional categories. By bridging human expression and machine interpretation, this feature ensures that diet logging becomes a quick, almost effortless task that users can perform anywhere and anytime. As a result, the system encourages consistent long-term usage, which is one of the most important success factors for any dietary monitoring platform.

### 4.3.3 Intelligent Food Matching

Once the LLM extracts structured data from a user’s meal description, the system performs intelligent food matching against a curated nutrition database to retrieve accurate nutrient information for each identified item. To handle the wide variation in how foods are spelled, abbreviated, or culturally named, the matching engine uses fuzzy string matching, which can recognize alternate forms of the same food, including small spelling differences, plural variations, and informal naming. This is especially important for regional foods such as “dal,” “daal,” or “dhal,” which represent the same item but appear in different forms in user input.

To strengthen this matching further, the system is designed to incorporate semantic similarity matching, allowing it to understand contextual relationships between food terms instead of relying purely on textual similarity. This enables it to map related items even when the exact name is not present in the database, improving accuracy and coverage especially for diverse cuisines. Finally, the matching engine assigns a confidence score to each matched item, and any item that falls below a defined confidence threshold is flagged for user confirmation. This human-in-the-loop verification mechanism ensures that ambiguous foods are never silently misclassified, helping the system maintain a strong balance between automation and accuracy.

### 4.3.4 Automated Nutrition Calculation

After foods are matched to the nutrition database, the system performs automated nutrition calculation to derive the complete nutritional profile of every meal. The system calculates calories as the primary measure of energy intake, allowing users to easily understand the overall impact of each meal on their daily targets. Beyond calories, it also computes the major macronutrients, including proteins, carbohydrates, and fats, which are essential for analyzing nutritional balance and aligning meals with health goals such as muscle building, weight management, or general wellness.

In addition to macronutrients, the system also calculates important supporting nutrients such as fiber content, which plays a critical role in digestive health, and sodium levels, which directly affect blood pressure and cardiovascular wellness. All these values are not only calculated for individual meals but are also aggregated into daily totals, giving users a clear and continuous understanding of their dietary intake. This automated process eliminates the need for manual nutritional bookkeeping and ensures that the user always has up-to-date and accurate nutrition data without any additional effort, allowing them to focus on their actual dietary decisions instead of complex calculations.

### 4.3.5 Personalized Recommendations

A defining strength of the system is its ability to generate personalized AI-driven recommendations that adapt to each user’s unique profile, preferences, and goals. Rather than producing generic dietary advice, the recommendation engine combines multiple personalization signals to create suggestions that genuinely fit the user’s lifestyle. It considers user profile attributes such as age, sex, weight, height, and activity level, all of which directly influence nutritional requirements and the type of guidance that is most appropriate for the individual.

In addition to demographic context, the recommendation engine also takes into account dietary preferences and restrictions, ensuring that suggestions remain compatible with vegetarian, vegan, low-carb, or other dietary lifestyles, and that no recommendation conflicts with declared allergies or intolerances. The system also factors in the user’s health goals, such as calorie targets, macronutrient splits, weight goals, and overall lifestyle objectives, to ensure that recommendations directly contribute toward the outcomes the user is working to achieve. To make recommendations even more relevant, the engine also analyzes historical eating patterns through a rolling seven-day window, allowing it to identify trends, deficiencies, or imbalances and respond with adaptive guidance instead of one-time generic advice. The result is a recommendation system that feels more like a personal nutrition coach than a static rule-based engine.

### 4.3.6 User Profile Management

The user profile management module forms the personalization backbone of the entire platform by collecting and maintaining the contextual information necessary for accurate and meaningful AI guidance. This module captures comprehensive demographic information, including age, sex, height, weight, and activity level, all of which are essential for calculating accurate calorie and macronutrient requirements. By centralizing these attributes within the profile, the system ensures that every other feature, from nutrition calculation to recommendation generation, operates with full awareness of the user’s individual context.

In addition to demographic data, the module also handles dietary preferences such as vegetarian, vegan, low-carb, high-protein, and other lifestyle-driven choices, allowing the system to consistently respect the user’s eating philosophy. It also stores allergies and intolerances as critical safety information, ensuring that any AI-generated suggestion always avoids items that could cause harm or discomfort. To make the platform truly inclusive, the module supports cultural food preferences, allowing users to indicate the cuisines and food traditions that match their daily eating habits, which significantly improves the relevance of recommendations and meal interpretations. Finally, the module enables users to define daily calorie and macronutrient targets, giving them a clear personal benchmark against which their dietary intake can be tracked and evaluated. Through this rich and well-structured personalization layer, the user profile management module ensures that the entire system continuously delivers context-aware, respectful, and goal-aligned dietary intelligence.

## 4.4 Data Flow

The Diet LLM Recommendation System processes every meal logging request through a clearly defined and tightly orchestrated data flow pipeline that connects user input to final personalized output. This pipeline ensures that conversational meal descriptions provided by the user are systematically transformed into structured, accurate, and meaningful dietary insights. Each stage of the pipeline performs a specialized role and passes its output as input to the next stage, allowing the entire system to operate as a continuous and intelligent processing chain.

The pipeline begins with the **user input** stage, where the user submits a natural language meal description through the application interface. This input is sent to the backend through a secure API request, marking the entry point of the entire data flow. Once the input reaches the backend, it moves into the **LLM parsing** stage, where the GPT-based language model interprets the conversational text and converts it into a structured JSON representation containing meal items, quantities, units, portions, and preparation indicators. This step is the foundational intelligence layer of the pipeline, as it bridges the gap between human language and machine-processable data.

After parsing, the data enters the **food matching** stage, where each parsed food name is compared against entries in the curated nutrition database. The matching engine handles spelling variations, regional names, and partial descriptions, ensuring that every recognized food maps accurately to the most relevant nutritional reference. Once matched, the system moves into the **nutrient calculation** stage, where the quantities provided by the LLM are mathematically combined with reference nutritional values to compute calories, macronutrients, and additional dietary components for each item, as well as totals for the entire meal.

Following nutrient calculation, the system enters the **database storage** stage, where the structured meal data, computed nutrient values, and associated metadata are persistently stored in the PostgreSQL database under the user’s profile. This stored data forms the historical foundation that powers personalization, dashboard insights, and trend analysis. The pipeline then progresses to the **recommendation engine** stage, where the system uses the user’s context, including profile information, dietary goals, preferences, and recent meal history, to generate AI-powered daily summaries and personalized recommendations. Finally, the pipeline concludes with the **response stage**, where the structured result, including parsed meal data, nutrient breakdown, recommendations, and confidence indicators, is returned to the client as a clean, predictable response, allowing the frontend to render dashboards, summaries, and feedback to the user. Through this end-to-end orchestrated flow, the system ensures that every user interaction is converted into accurate, personalized, and meaningful dietary intelligence in a fully automated manner.

## 4.5 Advantages of the Proposed System

The proposed Diet LLM Recommendation System offers a wide range of advantages that collectively transform the experience of dietary tracking and personalized nutrition guidance. The most significant advantage is the **dramatically improved user experience** delivered through natural language interaction. Users no longer need to navigate large food databases or memorize technical food names; instead, they simply describe their meals the way they would speak about them in everyday life. This conversational approach makes the system feel less like a tracking tool and more like a personal nutrition assistant, encouraging consistent long-term engagement.

A second major advantage is the **easy and intuitive interface** suitable for both technical and non-technical users. Because the system relies on natural language rather than structured input, it is equally accessible to users who may not be familiar with nutritional terminology or digital tracking platforms. This greatly broadens the potential user base, including older users, beginners, and users with limited technical exposure, all of whom can benefit equally from the platform. Closely related to this is the advantage of **faster meal logging**, where users can describe an entire meal in a single sentence instead of performing multiple manual searches, dramatically reducing the time and effort required for daily diet tracking.

The proposed system also offers **reduced user effort and minimized cognitive load**, since the AI takes over the tasks of parsing, matching, calculating, and analyzing dietary information automatically. This makes nutrition tracking sustainable in real-life conditions where users may not have the patience to interact with rigid forms repeatedly. Building on this, the system provides **personalized recommendations** that take into account the user’s health profile, dietary preferences, allergies, and individual goals, ensuring that no two users receive identical generic advice. Personalization is further enhanced by **better accuracy in dietary suggestions**, which is achieved through structured food data, validated nutrient calculations, and AI-driven contextual reasoning.

Another important advantage is the availability of **real-time nutrition insights**, including calorie counts, macronutrient breakdowns, and supporting dietary indicators, which become available immediately after each meal entry without requiring any manual computation. These real-time insights contribute strongly to **increased nutrition awareness**, as users gain a clear understanding of how each meal contributes to their overall daily intake. Over time, this awareness translates into healthier eating habits, balanced nutrition, and conscious dietary choices that align with long-term wellness goals.

The system also stands out due to its **strong support for diverse and culturally relevant food choices**, recognizing that real users come from a wide variety of culinary backgrounds. By interpreting regional food names, ingredients, and meal compositions, the system delivers a level of inclusivity that conventional diet trackers rarely achieve. Finally, the combination of all these advantages contributes to **higher user engagement and consistency in maintaining diet plans**, which is the most important success factor for any health-focused platform. Because the system feels intuitive, responsive, and personally relevant, users are more likely to continue using it regularly, ensuring that the long-term benefits of dietary awareness and AI-guided nutrition planning are actually realized in their daily lives. Together, these advantages position the proposed system as a clear improvement over traditional dietary tracking solutions and validate the strength of combining LLM intelligence with structured nutritional analysis in a single unified platform.
